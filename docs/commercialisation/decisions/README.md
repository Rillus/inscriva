# Architecture decision records (commercialisation)

Use this folder for **short ADRs** when choices are hard to reverse (auth vendor, entitlement token format, Stripe vs alternative, App Store vs direct sales).

## Naming

`ADR-0001-title-in-kebab-case.md`

## Template (copy into new file)

```markdown
# ADR-000N: Title

**Status:** Proposed | Accepted | Superseded  
**Date:** YYYY-MM-DD  
**Context:** …  
**Decision:** …  
**Consequences:** …  
```

When an ADR is accepted, link it from the relevant epic in [`../epics/`](../epics/) or from [`../implementation-plan.md`](../implementation-plan.md) §8.
